# Fortumhackathon
Deliverable for the Fortum Hackathon of team AUTO
The csv called 48hour_forecast_output.csv contains the predictions for the 48 hour forecasting.
The csv called 12month_forecast_output.csv contains the predictions for the 12 month forecasting.
The jupyter notebook called 48hour forecasting contains the notebook used to train the Sarima model and the resulting forecasting csv. The notebook was ran by Google Cloud computing using 96 vCPUs combining for 614 GB of RAM.
The jupyter notebook called 12month forecasting contains the notebook used to train the XGboosted model and the resulting forecasting csv. The notebook was ran locally.
The methodology provides a brief overview of why decisions were made and the process which has been done to get to the result.
The Fortum Energy Forecasting Pitch Deck provides a bit more visual representation of the methodology and the features of the project.
